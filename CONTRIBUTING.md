# Contribuindo

Obrigado pelo interesse em contribuir com o Iazeitor.

## Antes de mudar

- Abra uma issue antes de mudancas grandes.
- Descreva o problema e o impacto esperado.
- Confirme o escopo antes de iniciar.

## Branches

Use nomes curtos e objetivos:

- `feature/nome-curto`
- `fix/nome-curto`
- `docs/nome-curto`

## Pull Request

- Abra um Pull Request para revisao.
- Mantenha a mudanca dentro do escopo combinado.
- Garanta que o build passe antes de pedir revisao.

## O que nao commitar

- `bin`
- `obj`
- `.vs`
- dados reais
- tokens
- chaves
- logs sensiveis

## Commits

Use mensagens simples e descritivas, por exemplo:

- `ajustar parser de ações`
- `corrigir build da solucao`
- `atualizar documentacao`
